GET STARTED

Five-minute quickstart

Make your first request to the Mist API, then switch to your own key. Everything you need is on this page.

What you'll do

Send a chat completion request to Mist using a demo key, then swap in your own API key.

Prerequisites

  • A terminal with curl installed (or any HTTP client)
  • Optional: Python 3.9+, Node.js 18+, or Go 1.21+ if you prefer using an SDK
  • About five minutes

This quickstart uses a public demo key that allows 10 requests per IP per hour. For higher limits, contact sales.

1. Make your first request

Send the following request to the OpenAI-compatible chat completions endpoint. The demo key is rate-limited, but it is enough to verify connectivity and inspect the response format.

curl https://api.mist.llc/v1/chat/completions \
  -H "Authorization: Bearer mist_demo_pk_public_quickstart" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "auto",
    "messages": [
      {"role": "user", "content": "In one sentence, what is Mist?"}
    ]
  }'

When "model" is set to "auto", Mist picks the cheapest capable model for the request. You can also pass a specific model name like "gpt-4o-mini" or "claude-3-5-sonnet" — see the routing guide for details.

2. Understand the response

The response uses a familiar chat completions envelope. Notice that the returned model is a concrete provider model even though the request asked for "auto".

{
  "id": "req_01hzk8m7r4w2n9d5c6b3a1f0e",
  "object": "chat.completion",
  "created": 1762483200,
  "model": "gpt-4o-mini",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Mist is an API gateway that routes AI requests across multiple model providers through one OpenAI-compatible endpoint."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 17,
    "completion_tokens": 22,
    "total_tokens": 39
  }
}
  • id — unique request identifier; include this when contacting support
  • model — the actual model that served the request (not necessarily what you asked for, when using "auto")
  • choices[0].message.content — the model output
  • usage — token counts; this is what you'll be billed for
  • x-mist-request-id (header) — same as id, also returned in response headers

3. Use your own API key

Replace the demo key with your production key once you are ready to test under your own account. Production keys are issued by sales; contact Mist to obtain one. API keys look like mist_live_pk_... for production and mist_test_pk_... for the test environment.

curl https://api.mist.llc/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "auto",
    "messages": [
      {"role": "user", "content": "In one sentence, what is Mist?"}
    ]
  }'

4. Install an SDK (optional)

Python

pip install mist
from mist import Mist
client = Mist(api_key="mist_live_pk_...")
response = client.chat.completions.create(model="auto", messages=[{"role":"user","content":"hello"}])
print(response.choices[0].message.content)

Node.js

npm install @mist/sdk
import Mist from "@mist/sdk";
const client = new Mist({ apiKey: "mist_live_pk_..." });
const response = await client.chat.completions.create({ model: "auto", messages: [{ role: "user", content: "hello" }] });
console.log(response.choices[0].message.content);

Go

go get mist.llc/mist-go
client := mist.NewClient("mist_live_pk_...")
response, err := client.Chat.Completions.Create(
    context.Background(),
    mist.ChatCompletionRequest{Model: "auto", Messages: []mist.Message{{Role: "user", Content: "hello"}}},
)
fmt.Println(response.Choices[0].Message.Content)

5. Migrating from OpenAI?

Because Mist is OpenAI-compatible, you can use the official OpenAI SDK and just change the base URL.

OpenAI(base_url="https://api.mist.llc/v1", api_key="mist_live_pk_...")

This is the fastest path to production for teams already using OpenAI directly.
