Mist developer documentation

Everything you need to integrate Mist into your application, from a five-minute quickstart to the full API reference.

Mist is an AI orchestration gateway for teams that want one production API instead of managing direct integrations with multiple model vendors. You send requests to a single OpenAI-compatible endpoint, and Mist routes them across 20+ providers including OpenAI, Anthropic, Google, and DeepSeek. The platform gives you one API key, one bill, one SLA, and provider failover when an upstream service is slow or unavailable.

For most integrations, existing code keeps working with a single base_url change. Mist preserves the OpenAI API shape on the client side, while adding controls that are difficult to build in-house: routing by task type, real-time spend caps, audit logs, and optional Prism verification for responses that need additional confidence. Start with the quickstart if you want to make a request now, or read the concepts pages first if you need to understand how routing and verification behave in production.

What is Mist?

Mist is an AI orchestration gateway, not a model. It sits between your application and the underlying model providers. On the front end, it speaks the OpenAI API, so existing OpenAI clients and most existing request payloads work without application-level rewrites.

On the back end, Mist routes each request to the cheapest capable model based on a task-type hint, falls back to alternates when a provider degrades, and records every request in an audit trail. For most engineering teams, the practical benefits are straightforward: lower model spend, typically in the 30–60% range; zero vendor lock-in because no provider-specific client is required; and built-in failover without maintaining custom routing logic yourself.
