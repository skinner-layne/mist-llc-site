REFERENCE

API reference

Mist exposes an OpenAI-compatible REST API. This page covers authentication and the chat completions endpoint. Additional endpoints will be documented as they ship.

Base URL

The production base URL is https://api.mist.llc/v1. The test environment is https://api-test.mist.llc/v1. All endpoints in this reference are relative to that base.

Authentication

Authenticate every request with a bearer token in the Authorization header: Authorization: Bearer YOUR_API_KEY. Production keys use the prefix mist_live_pk_.... Test keys use the prefix mist_test_pk_....

API keys are provisioned by sales and can be rotated through the dashboard. Use separate keys for production and test environments to keep traffic and billing boundaries clear.

Security

Never embed API keys in client-side code or public repositories. All API access should be server to server.

Request format

Requests use JSON over HTTPS. Unless noted otherwise, send the following headers with each request.

Header Required Value Description
Authorization Yes Bearer YOUR_API_KEY Bearer token for the target environment.
Content-Type Yes application/json Required for JSON request bodies.
Idempotency-Key No string Optional key, maximum 256 characters. Use a UUID. Replays within 24 hours return the cached response.
X-Request-ID No string Caller-supplied request ID. If omitted, Mist generates one.

Response format

Every response is a JSON object. Successful responses follow the OpenAI response envelope. Errors return a JSON object with an error field. Standard response headers include X-Mist-Request-ID, X-Mist-Model (the actual model that served the request), X-Mist-Latency-Ms, and X-RateLimit-Remaining.

Endpoints

POST /v1/chat/completions

Generate a chat completion. OpenAI-compatible — drop-in replacement for the OpenAI chat.completions.create endpoint, with optional Mist-specific fields for routing and verification.

Request body

Field Type Required Description
model string Yes Model identifier. Use "auto" to let Mist pick. Pass a specific model name (for example, "gpt-4o-mini", "claude-3-5-sonnet") to pin to one provider.
messages array Yes List of message objects with role (system|user|assistant) and content (string).
task_type string No Mist-specific routing hint. One of: reasoning, code, creative, summarization, quick_response, etc. See routing.
temperature number No Sampling temperature, 0.0–2.0. Default 1.0.
max_tokens integer No Maximum tokens in the response.
stream boolean No If true, returns a server-sent event stream.
verification string No Mist-specific. One of: none (default), single, prism. See verification.
fallback_chain array No Mist-specific. Ordered list of model names to try if the first fails.

Example request

curl https://api.mist.llc/v1/chat/completions \
  -H "Authorization: Bearer mist_live_pk_..." \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: 6b1dcfa3-1f46-4bdb-a0c5-2a0bc2e6d3f7" \
  -d '{
    "model": "auto",
    "messages": [
      {
        "role": "user",
        "content": "Explain why connection pooling matters for high-throughput APIs."
      }
    ],
    "task_type": "reasoning",
    "temperature": 0.2,
    "max_tokens": 300
  }'

Response

Field Type Description
id string Unique identifier for the completion.
object string Always "chat.completion".
created integer Unix timestamp in seconds.
model string The model that actually served the request.
choices array Array of completion choices containing message objects and finish reasons.
usage object Token accounting with prompt_tokens, completion_tokens, and total_tokens.
mist_metadata object Mist-specific metadata containing request_id, routed_to, latency_ms, and cost_usd.

Errors

  • 400 invalid_request — malformed body or unknown model
  • 401 unauthorized — missing or invalid API key
  • 429 rate_limit_exceeded — your account exceeded its per-minute or per-day quota; retry after the time in the Retry-After header
  • 503 upstream_unavailable — every model in the fallback chain failed; retry the request

Errors

Global errors use a consistent envelope.

{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Your account exceeded 60 requests per minute on this endpoint.",
    "request_id": "req_01HF...",
    "retry_after_ms": 1200
  }
}

Always log error.request_id when handling failures. That ID is the fastest way for Mist support to locate the request in internal logs. If the error includes retry_after_ms or a Retry-After header, use that value rather than a fixed retry interval. For high-volume integrations, treat 429 and 503 as retryable and apply backoff with jitter.
